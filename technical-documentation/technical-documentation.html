Introduction

Git is a version controlling software, which allows us to track progress of files or projects over the time. Git allows us to distribute changes over the time chronologically.

Git is not first Version Controlling Software (VCS), there were mamy software available like BitKeeper, Mercurial, Monotone, CVS, SVN etc.

A short history of Git

Git was designed by Linus Torvald, who designed Linux kernel.

In 2005, the relationship between commuity that developed Linux kernel and the commertial company that devloped version controlling software (BitKeeper) broke down. The tool's status was free of charge which was revoked. This is the reason why Git was developed.

Goals of the new version controlling system wre asfollows:

  • speed
  • Simple Design
  • Strong support for non lineardevelopment
  • Fully distributed
  • Able to handle large projects.

Advantages of Git

  • Prevent against data loss.
  • Manage complex projects.
  • Distribute your file changes over the time.
  • Distibuted repository model.
  • Compatible with existing systems and protocol.

Installation

Git can be installed on any operating syatems. Here we will see the installation process of Git in Ubuntu 18.04.

Step 1:

$ sudo apt update

Step 2:

$ sudo apt install git

Now Git has been installed, to verify check version of Git.

Step 3:

$ git --version

Output of command mentioned above will show you installed version of Git.

Configure Git

Step 4: Configure user name

$ git config --global user.name "user_name"

Step 5: Configure email address

$ git config --global user.email "your_email"

Step 6: Editing configuring file

$ nano ~/.gitconfig

This step is used to configure user name and email address.

Repositories

Git repository contains collection of files and different versions of project. This repository tracks all the changes made to files in your project, building history over time.

Repositories are also known as 'repos'. Reositories stored on local machine are called aas local repositories and repositories stored on server are called as remote repositories.

Repository vs Directories

A repository include all previous changes whereas directories store projects at their current state in time.

Creating a repository

There are two ways to create Git repositories.

  1. Import existing project into Git.
  2. Clone an existing Git repository from another server.

Import existing project into Git

Step 1: Goto project folder, right click on empty area select "open in terminal" OR Open terminal and move to project folder.

Step 2: After moving to project folder from terminal type following command

$ git init

NOTE: This step is irreversible.

When above mentioned command is executed ".git" folder is created in your project folder which store all information/status of your project.

Step 3: Moving to staging area. Use following command.

$ git status

To add files in staging area use following command.

$ git add "file-name" OR

$ git add .

The first command will add a single file to staging area.

The second command will add multiple files to staging area.

If you want to unstage or remove a file from staging area then

$ git rm -f "file-name" OR

$ git rm "file-name"

It deletes file from project permenantly

If you want to keep a file in a project but do not want to track then

$ git rm --cashed "file-name"

Step 4: Commit

$ git commit -m OR

$ git commit -a -m "message"

After this step all the files are commited.

$ git log

This command shows all the commit done by you on your project.

Clone an existing Git repository from another server

$ git clone

This command receives full copy of all data that the server has. The "URL" includes a directory name and all files of project. If you want to clone repository in otherfolder then

$ git clone "url" "folder-name"

The three states of Git

Git has three main states that your files can resides in: commited, modified, staged.

Committed: Data is safely stored in local database.

Modified: You have changed the file but have not committed it to your database yet.

Staged: You have marked a modified files inits current version to go into your next commit snapshot.

Main sections of Git Project

Git Directory

The Git directory stroes metadata and object database foryour project. Git directory is copied when you clone a repository from another computer.

Working Directory

Files in working directory are pulled out of the compressed database in the git directory and placed on disk for you to use or modify.

Staging Area

Staging area is a file, generally contained in your git directory. It stores the information about what will go into your next commit.

Git Branches

Git branches are a way to create separate development path without overriding or creating copies of your project.

Branches can be added, deleted and merged. Tahe main branch is known as "master". Master branch is created when you commit for first time.

Working of a Git branch

When you make a "commit" Git stores commit object. This object contain name and email of author, message that you typed and a pointer to the commit. When we stage a file each file is checksum, stores that version of file in git repository. Git refer to this as blob and add checksum to staging area.

When we commit, Git checksums each subdirectory and stores tree object in Git repository. Git then creates commit object which contains metadata and pointer to the root project tree so it can re-create that snapshot when needed.

Git repository now comtains 1 blob for the content of each of your files, 1 tree that list the content of directory and specifies which files are atored as which blob and 1 commit with pointer to root tree and all the commit metadata.

Creating new branch.

Before creating a new branch in your project development you must have commit first time. This initial commit will create master branch. Now we will create

$ git branch "name-f-branch" OR

$ git checkout -b "name-of-branch"

First command creates a new branch. Second command creates and switches to newly creates branch.

Switching back to master branch

$ git checkout master

Listing all the branches of your project.

$ git branch -a

-a = all branches and* = indicates branch you are currently on.

Deleting a branch.

$ git branch -d "name-of-branch"

Merging a branch

$ git merge "message/branch-name"

This process is difficult sometimes. Is merging fails then resolve each file manually.

$ git add "file-name"

Staging a file marks it as resolved on Git. You can use graphical tool instead of command line.

$ git mergetool

Use following command to list which branches are merged in branch you are currently on.

$ git branch --merged and

$ git --nomerged To list which branches that haven't merged yet.
